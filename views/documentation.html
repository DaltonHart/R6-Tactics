Getting Started

Introduction

Welcome to the Rainbow Six Siege API! This documentation should help you familiarise yourself with the resources available and how to consume them with HTTP requests. Read through the getting started section before you dive in. Most of your problems should be solved just by reading through it.

Base Url

The Base URL is the root URL for all of the API, if you ever make a request and you get back a 404 NOT FOUND response then check the Base URL first.

The base url for this api is:

localhost:3000/api

Authentication

This is a completely open API. No authentication is required to query and get data. This also means that we've limited what you can do to just GET requests. If you find a mistake in the code base feel free to email the developer. Contact Here

Rate Limiting

Rate limiting has been set to prevent malicious abuse and to make sure our service can handle a potentially large amount of traffic. Rate limiting is done via IP address and is currently limited to 10,000 API request per day.

Operators

There are two operator endpoints for organization.

Attack Operators

The Attack Operator Endpoints are defined as:

  • /atkop

    Will return all Attack Operators.

  • /atkop/:id

    Will return a specific Attack Operator with the given Id.

Example Request:

GET localhost:3000/api/atkop

Example Response:

Attributes
  • name - String - The name of the operator.
  • _id - String - Database ID of the specific operator.
  • ability - Object - Ability is an object refrenced from the Ability endpoint.
  • desc - String - A description of the operator.
  • badge - String - Url endpoint for the operator badge.
  • org - String - Organization the operator belongs to.
  • img - String - Url endpoint for hte operator image.
  • bio - String - Background story of the operator.
  • armor_rating - Number - Numerical rating of armor.
  • speed_rating - Number - Numerical rating of speed.
  • primary_weapons - Array of Objects - Array of primary weapon objects available for selection for the operator. Weapons are refrenced from the Weapons endpoint.
  • secondary_weapons - Array of Objects - Array of secondary weapon objects available for selection for the operator. Weapons are refrenced from the Weapons endpoint.
  • gadgets - Array of Objects - Array of gadget objects available for selection for the operator. Gadgets are refrenced from the gadgets endpoint.

Abilities

The operator ability endpoints are defined as:

  • /abilities

    Will return all operator abilities.

  • /abilities/:id

    Will return a specific operator ability with the given Id.

Example Request:

GET localhost:3000/api/abilities/1

Example Response:

Attributes
  • name - String - The name of the operator ability.
  • _id - String - Database ID of the specific ability.
  • uses - Number - Numerical value of the times an ability can be used.
  • desc - String - A description of the operator ability.
  • img - String - Url endpoint for the operator ability image.

Gadgets

The operator gadget endpoints are defined as:

  • /gadgets

    Will return all operator gadgets.

  • /gadgets/:id

    Will return a specific operator gadget with the given Id.

Example Request:

GET localhost:3000/api/gadgets/1

Example Response:

Attributes
  • name - String - The name of the operator gadget.
  • _id - String - Database ID of the specific gadget.
  • uses - Number - Numerical value of the times a gadget can be used.
  • desc - String - A description of the operator gadget.
  • img - String - Url endpoint for the operator gadget image.
  • assignment - String - Shows assignment to either attack operators or defense operators.

Weapons

The operator weapon endpoints are defined as:

  • /weapons

    Will return all operator weapons.

  • /weapons/:id

    Will return a specific operator weapon with the given Id.

Example Request:

GET localhost:3000/api/weapons/1

Example Response:

Attributes
  • name - String - The name of the operator weapon.
  • _id - String - Database ID of the specific weapon.
  • img - String - Url endpoint for the operator weapon image.
  • type - String - The type of weapon.
  • assignment - String - Shows assignment to either a primary weapon slot or secondary weapon slot.
  • firemodes - Array - An array of strings for the available firemodes.
  • sights - Array - An array of strings for the available sight attachments.
  • barrels - Array - An array of strings for the available barrel attachments.
  • grips - Array - An array of strings for the available grip attachments.
  • underbarrel - Array - An array of strings for the available underbarrel attachments.
